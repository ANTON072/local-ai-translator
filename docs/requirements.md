# Local MS Translator

Mac 上で、選択した英文を `cmd + j` で即座に日本語へ翻訳するローカルアプリケーション。
英語 → 日本語のみ対応。自分専用・このMacのみで利用し、一般公開はしない。
翻訳はローカルLLM（Ollama）で行い、テキストを社外へ送信しない完全ローカル動作。

## ユースケース

1. アプリを起動・常駐させておく。
2. ブラウザや Teams で英語のテキストを選択し、`cmd + j` を押す。
3. アプリが Spotlight 風に画面中央上へ表示され、選択テキストを取得して
   ローカルLLM（Ollama）で翻訳する。
4. アプリ上に日本語訳が表示される。
5. 結果はコピーボタンを押したときだけクリップボードにコピーされる。
6. 閉じるボタン、または再度 `cmd + j` でアプリが非表示になり、入力内容はリセットされる。
7. アプリのテキストフィールドに英語を直接入力して翻訳ボタンを押しても、同じ翻訳挙動になる。

## 選択テキストの取得方式

- グローバルショートカット `cmd + j` 押下時に、`cmd + C` を擬似的に送信して
  選択テキストをクリップボード経由で取得する。
- 取得前にユーザーの既存クリップボード内容を退避し、取得後に復元する
  （ユーザーのクリップボードを壊さない）。
- キー擬似入力には macOS の **アクセシビリティ権限** が必要。
  初回起動時、権限が未許可ならガイド画面を表示してシステム設定へ誘導する。

## ウィンドウ仕様

- Spotlight 風：フレームレス、画面中央やや上、常に最前面。幅 720px・横長で、
  訳文量に応じて高さを自動伸長（上限ありでスクロール）。
- 表示／非表示のトグルは `cmd + j`。閉じるボタンでも非表示。
- 非表示時に入力・結果をリセットする。
- メニューバー（トレイ）にアイコンを常駐させ、設定・終了を出す。Dock には出さない。
- 訳文はストリーミングで逐次表示する。

## 翻訳の挙動

- 起動時にモデルを先読みロードし、初回翻訳のロード待ちを抑える（`keep_alive` 長め）。
- `cmd + j` 時に選択テキストが空なら、エラーにせず手動入力モードで表示する。
- リクエストは 30 秒でタイムアウト、入力は 5000 文字で打ち切る。

## 設定

- 翻訳に使う Ollama のモデル名（既定 `qwen2.5:14b`）とエンドポイント
  （既定 `http://localhost:11434`）を `~/.config/local-ms-translator/config.json` に保存する。
- アプリ内の設定画面から入力・保存できる。
- 翻訳の呼び出しはアプリのバックエンド（Rust）側で行う（秘匿情報は持たない）。

## 技術スタック

- Tauri v2（バックエンド: Rust）
- フロントエンド: TypeScript / React / shadcn/ui
- 翻訳: ローカルLLM（Ollama、既定 `qwen2.5:14b`）。`/api/chat` へ POST し英→日翻訳。
- 対象: macOS のみ。ソースは GitHub の private リポジトリ。

## 非対象 / 前提

- 英→日のみ（他言語・双方向は対象外）。
- 完全ローカル動作（翻訳テキストは社外へ送信しない）。Ollama の常駐と
  モデルの事前 pull が前提。ネット接続が無くても翻訳できる。
- 配布・公開・複数ユーザー対応はしない。
